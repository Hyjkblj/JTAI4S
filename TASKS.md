# XtalLoop 任务清单

> 目标：优先证明“飞书会议/妙记 -> 结构化研发知识 -> 人工审核 -> Base/任务/知识库 -> Obsidian 复用”闭环真实可运行，再补齐工程化、评测与参赛材料。

## 0. 状态与原则

### 当前状态

- [x] 完成产品 PRD、架构、数据模型、指标和 12 周路线设计
- [x] 核验飞书 CLI 官方能力范围
- [x] 初始化 Git 仓库，默认分支为 `main`
- [ ] 建立可运行原型
- [ ] 建立脱敏数据黄金集和评测报告
- [ ] 完成海选附件与公开链接

### 优先级

- **P0**：海选竞争力和核心可行性必需，必须优先完成。
- **P1**：形成完整产品体验和企业试点能力，P0 跑通后推进。
- **P2**：规模化、深度本体和受监管能力，不阻塞当前参赛。

### 执行原则

1. 先验证真实接口和权限，再开发上层功能。
2. 所有演示结论必须能回到会议片段、说话人和时间戳。
3. 关键数值、单位、风险和最终决策必须人工审核。
4. 飞书保存组织级事实，Obsidian 保存授权摘要和个人批注，图谱是可重建索引。
5. 没有真实实验数据时使用明确标注的合成数据，不虚构企业实测效果。
6. 不将 App Secret、Token、真实会议内容或敏感研发数据提交到 Git。

**验收标准：**

- 任何写操作先完成 `--dry-run` 或等价预览。

**Go/No-Go：** 若不能合法取得带来源定位的会议文本，暂停自动证据链开发，改用脱敏导入样本完成比赛原型，并在材料中明确接口依赖。

## 5. 比赛版 Definition of Done

- [ ] 一条脱敏会议闭环真实运行，而不是仅展示静态页面
- [ ] 所有关键结论都能回到说话人、时间戳和原文证据
- [ ] 关键参数经过人工审核后才发布
- [ ] 飞书 CLI 权限和失败边界有实测记录
- [ ] Obsidian 展示的是授权知识卡，不形成第二事实源
- [ ] 至少一个历史失败案例成功影响新实验方案
- [ ] 有黄金集、评测指标和错误案例，不只展示成功样例
- [ ] GitHub、视频、PDF 和项目主页无需登录即可访问
- [ ] 仓库中不存在密钥、真实研发数据和个人敏感信息
- [ ] 所有业务价值指标均标记为试点目标或实测结果，不虚构收益
